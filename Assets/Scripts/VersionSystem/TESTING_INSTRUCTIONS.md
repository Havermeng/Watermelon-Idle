# Инструкция по тестированию системы версий

## 🧪 Быстрый старт тестирования

### 1. Подготовка сцены

1. Откройте сцену **MainMenu** (или любую другую сцену с Canvas)
2. Убедитесь, что в сцене есть **Canvas** объект

### 2. Добавление системы

**Способ A: Полностью автоматический (рекомендуется)**

1. Создайте пустой GameObject в сцене
2. Назовите его `VersionUIManager`
3. Добавьте компонент `VersionUIManager` (скрипт: `Assets/Scripts/VersionSystem/VersionUIManager.cs`)
4. В инспекторе проверьте настройки:
   - ✅ `Create UI Automatically` = true
   - ✅ `Show Version Button` = true
   - `Version Button Position` = (100, 50) - можно изменить
   - `Version Panel Size` = (600, 500) - можно изменить

5. Запустите сцену (Play Mode)

**Что должно произойти:**
- Автоматически создастся панель версий (VersionPanel)
- Автоматически создастся кнопка (VersionButton)
- При запуске проверится версия и покажется уведомление (если версия новая)

---

**Способ B: Ручная настройка (полный контроль)**

1. **Добавьте VersionManager:**
   - GameObject → `VersionManager`
   - Component → `VersionManager`
   - В поле `Current Version` установите `0.1.3` (или любое другое)
   - `Use JSON File` = true
   - `Versions JSON File` = перетащите `versions.json` из папки Resources

2. **Добавьте VersionUIManager:**
   - GameObject → `VersionUIManager`
   - Component → `VersionUIManager`
   - `Create UI Automatically` = false (если хотите создать UI вручную)
   - `Show Version Button` = false (если создаете кнопку вручную)

3. **Добавьте VersionNotifier:**
   - GameObject → `VersionNotifier`
   - Component → `VersionNotifier`
   - `Show On Start` = true
   - `Notification Duration` = 8 (секунд)

4. **Создайте UI вручную (если не используете автоматическое):**
   
   **Панель версий:**
   - В Canvas создайте Panel → назовите `VersionPanel`
   - Добавьте компонент `VersionPanelUI`
   - Добавьте ScrollView внутрь панели
   - В ScrollView → Viewport → Content создайте TextMeshPro Text
   - Назначьте этот Text в поле `Version Text` компонента VersionPanelUI
   
   **Кнопка:**
   - В Canvas создайте Button → назовите `VersionButton`
   - Добавьте компонент `VersionButton`
   - В поле `Version Panel` перетащите `VersionPanel`
   - В инспекторе кнопки в OnClick() добавьте: `VersionButton → OpenVersionPanel()`

5. **Добавьте кнопку в MenuManager:**
   - Откройте `MenuManager` в инспекторе
   - В поле `Versions Button Text` назначьте TextMeshPro текст для кнопки
   - В коде MenuManager уже есть метод `OpenVersions()`
   - Настройте кнопку в UI: OnClick() → `MenuManager → OpenVersions()`

6. Запустите сцену (Play Mode)

### 3. Тестовые проверки

#### ✅ Проверка 1: Загрузка версий

Добавьте тестовый скрипт `VersionSystemTest` на любой GameObject:

```csharp
// В инспекторе нажмите на компонент и используйте кнопки в Inspector
// Или вызовите методы из другого скрипта:
VersionSystemTest test = FindObjectOfType<VersionSystemTest>();
test.TestCheckVersion();
```

**Ожидаемый результат в Console:**
```
=== VersionSystemTest Started ===
VersionManager found. Current version: 0.1.3
Total versions loaded: 4
  - v0.1.3: Добавлена система истории версий...
  - v0.1.2: Добавлена система локализации...
  - v0.1.1: Исправлен баланс...
  - v0.1.0: Добавлены арбузы...
VersionUIManager found and initialized.
VersionNotifier found and initialized.
Is new version: true/false
=== VersionSystemTest Complete ===
```

---

#### ✅ Проверка 2: Открытие панели версий

**Вручную:**
- Нажмите на кнопку "📋 Версии"
- Должна открыться панель с анимацией
- В панели должен быть виден список всех версий
- Текущая версия должна быть помечена "✓ Текущая"

**Через код:**
```csharp
VersionUIManager.Instance.ShowVersionPanel();
```

---

#### ✅ Проверка 3: Уведомление об обновлении

**Тест 1: Первый запуск (новая версия)**

1. Удалите PlayerPrefs:
   ```csharp
   PlayerPrefs.DeleteKey("LastGameVersion");
   PlayerPrefs.Save();
   ```
   Или используйте `TestResetSavedVersion()` из VersionSystemTest

2. Перезапустите сцену (Stop → Play)

**Ожидаемый результат:**
- В верхней части экрана появится уведомление с анимацией
- Уведомление содержит:
  - Заголовок: "🎉 Обновление!"
  - Сообщение: "Доступна новая версия 0.1.3"
  - Кнопка "📋 Посмотреть"
  - Кнопка "✕ Позже"
- Через 8 секунд уведомление исчезнет (если не нажать на кнопки)

**Тест 2: Нажатие "Посмотреть"**
- Нажмите "📋 Посмотреть"
- Уведомление должно закрыться с анимацией
- Должна открыться панель версий

**Тест 3: Повторный запуск**
- Не закрывая сцену, нажмите Stop
- Снова нажмите Play
- Уведомление НЕ должно появиться (версия уже сохранена)

---

#### ✅ Проверка 4: Локализация

1. Убедитесь, что `LocalizationManager` работает
2. Измените язык через LocalizationManager
3. Текст кнопки "📋 Версии" должен обновиться
4. Если добавить ключи в localization, уведомление тоже должно переводиться

---

#### ✅ Проверка 5: Анимации

**Панель:**
- При открытии: плавное увеличение от 0.8x до 1x + fade in
- При закрытии: плавное уменьшение до 0.8x + fade out
- Длительность: Open = 0.3с, Close = 0.2с

**Уведомление:**
- Появление: scale от 0 до 1 за 0.3с
- Исчезновение: scale от 1 до 0 за 0.2с

### 5. Отладка常见问题

**Панель не создается:**
- Проверьте консоль на ошибки "No Canvas found"
- Убедитесь, что Canvas есть на сцене
- Проверьте `Create UI Automatically` = true

**Кнопка не работает:**
- Проверьте, что кнопка имеет компонент Button
- Проверьте OnClick() в инспекторе кнопки
- В консоли должно быть: "VersionButton: Button component not found" если кнопка без Button

**Уведомление не появляется:**
- Проверьте `VersionNotifier.Instance` в консоли
- Проверьте `VersionManager.Instance.IsNewVersion()` - должно быть true
- Убедитесь, что `showOnStart` = true
- Проверьте, что `hasBeenShown` = false (сбросьте через `ResetNotificationFlag()`)

**Текст не отображается:**
- Убедитесь, что TextMeshPro установлен (Window → Package Manager)
- Проверьте, что в панели есть TextMeshProUGUI компонент
- Проверьте цвет текста (должен быть белым на темном фоне)

**JSON не загружается:**
- Убедитесь, что `versions.json` находится в `Assets/Resources/`
- Проверьте структуру JSON (должна быть {"versions": [...]})
- В консоли должно быть: "Loaded X versions from JSON"

### 6. Тестирование в сборке

1. Соберите игру (File → Build Settings → Build)
2. Запустите собранную версию
3. Проверьте:
   - Уведомление при первом запуске
   - Открытие панели через кнопку
   - Скролл текста
   - Закрытие панели

---

### 7. Чистый тест (сброс всех данных)

Для полного теста с нуля:

1. Закройте Unity
2. Удалите папку `Library/` (необязательно, но поможет)
3. Удалите PlayerPrefs:
   - Windows: `HKEY_CURRENT_USER\Software\Unity\UnityEditor\...` (в реестре)
   - Или в коде: `PlayerPrefs.DeleteAll();`
4. Откройте проект заново
5. Запустите сцену
6. Все должно создаться с нуля

## 🎯 Критерии успешного теста

- [ ] Все скрипты компилируются без ошибок
- [ ] VersionManager загружает версии из JSON
- [ ] Панель версий создается (автоматически или вручную)
- [ ] Кнопка "Версии" отображается и работает
- [ ] При нажатии на кнопку открывается панель
- [ ] В панели отображаются все версии в обратном порядке (новые сверху)
- [ ] Текущая версия помечена специальным значком
- [ ] Уведомление появляется при первой установке/обновлении
- [ ] Уведомление имеет анимацию появления/исчезновения
- [ ] Кнопка "Посмотреть" открывает панель версий
- [ ] Уведомление автоматически скрывается через N секунд
- [ ] После закрытия уведомления версия сохраняется в PlayerPrefs
- [ ] При повторном запуске уведомление не показывается
- [ ] Локализация работает (если есть LocalizationManager)
